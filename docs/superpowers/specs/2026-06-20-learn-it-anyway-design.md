# Learn It Anyway — Blog Design Spec

**Date:** 2026-06-20
**Author:** Navneet Mahajan
**Status:** Approved

---

## Overview

**Learn It Anyway** is a personal learning journal and digital garden by Navneet Mahajan. It documents curiosity in public — primarily around software architecture, systems, AI, databases, and DevOps, with a dedicated corner for books, life lessons, and mental models ("Side Quests").

**Core motto:** Learn before you need it.

---

## Stack

| Layer | Choice | Reason |
|---|---|---|
| Framework | Astro | Purpose-built for content sites, ships zero JS by default, simple file-based routing |
| Styling | Tailwind CSS | Fast to customize, excellent `prose` plugin for blog text |
| Content | Markdown (`.md`) + Astro Content Collections | Type-safe frontmatter, simple daily workflow |
| Hosting | Vercel | GitHub auto-deploy, free tier, ~30s deploys |
| Version control | GitHub | Push to deploy trigger |

---

## Post Frontmatter

```yaml
---
title: "How Databases Actually Work"
date: 2026-06-20
description: "Past the SQL prompt, a database is a story about durability, ordering, and trust."
tags: ["systems", "databases"]
---
```

All fields required. `tags` is an array. `date` is ISO 8601 (`YYYY-MM-DD`).

---

## Daily Publishing Workflow

1. Create a new `.md` file in `src/content/articles/` or `src/content/side-quests/`
2. Filename format: `YYYY-MM-DD-slug.md`
3. Add frontmatter (title, date, description, tags)
4. Write post in Markdown (supports code blocks, images via public/ folder)
5. `git add . && git commit -m "post: <title>" && git push`
6. Vercel auto-deploys — live in ~30 seconds
